# README 업데이트 작업

README.md 파일에 오늘 날짜의 작업 내역을 업데이트해주세요.

## 작업 순서

1. 먼저 `git log -50 --pretty=format:"%h %ai %s"` 명령어로 최근 50개의 커밋 내역을 확인한 후, 오늘 날짜(YYYY-MM-DD 형식)에 해당하는 커밋만 필터링
   - **중요**: --since 옵션은 시간대 문제로 오늘 커밋을 놓칠 수 있으므로, 반드시 최근 커밋 목록을 먼저 확인한 후 날짜로 필터링할 것
   - **요일 계산**: 해당 날짜의 정확한 요일을 확인하려면 다음 명령어 사용:
     - macOS: `date -j -f "%Y-%m-%d" "YYYY-MM-DD" +"%u"` (1=월요일, 2=화요일, 3=수요일, 4=목요일, 5=금요일, 6=토요일, 7=일요일)
     - 또는 `date -j -f "%Y-%m-%d" "YYYY-MM-DD" +"%A"`로 영문 요일 확인 후 한글로 변환
   - **필수**: 날짜와 요일이 정확히 일치하는지 반드시 확인하고, 잘못된 경우 수정할 것
2. 커밋 메시지들을 카테고리별로 분류:
   - API 연동 및 기능 추가
   - UI/UX 개선
   - 로그인 관련 기능
   - 버그 수정
   - 기타
3. README.md의 "## 일별 작업 내역" 섹션 최상단에 오늘 날짜(YYYY-MM-DD 요일 형식)로 새 섹션 추가
4. 분류된 내역을 적절한 제목(####)과 함께 bullet point로 작성
5. 구분선(---)으로 이전 날짜와 구분

## 주의사항

- 커밋 메시지의 prefix([kakaoMapAPI], [feature] 등)는 제거하고 내용만 정리
- 유사한 작업은 하나로 통합
- 기술적으로 의미있는 작업만 포함
- 날짜 형식: "### YYYY-MM-DD (요일)"
- Merge PR 커밋은 PR 번호와 함께 메인 카테고리 제목으로 사용

## 예시

만약 오늘(2025-12-02)의 커밋이 다음과 같다면:
```
8f83260 2025-12-02 01:37:15 +0900 [emailCheck] feat: 유저 데이터 생성 훅 추가
084e01f 2025-12-02 01:31:22 +0900 [emailCheck] fix: fetching API 테이블 명칭 수정
```

**요일 확인**: `date -j -f "%Y-%m-%d" "2025-12-02" +"%u"` → 결과: 2 (화요일)

README.md에는 다음과 같이 작성:
```markdown
### 2025-12-02 (화)

#### 기능 추가
- 유저 데이터 생성 훅 추가

#### 버그 수정
- fetching API 테이블 명칭 수정

---
```
